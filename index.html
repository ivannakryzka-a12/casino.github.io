2. Загальний опис

2.1 Основні функціональні блоки

  • Управління акаунтом: реєстрація, автентифікація, налаштування профілю
  • Беттинг: перегляд подій, розміщення та зміна ставок, отримання результатів
  • Фінанси: купівля токенів, поповнення рахунку, виведення коштів
  • Соціальні функції: чат, список друзів, слідкування за іншими гравцями
  • Контент: стрічка спортивних новин, статистика ставок
  • ШІ-асистент (бажана функція): персональні рекомендації ставок

2.2 Типи акторів та їх ролі

  • Гість - перегляд публічного контенту (новини, загальна статистика)
  • Зареєстрований (Standard) - повний доступ до беттингу та базових функцій
  • Преміум-користувач - розширені функції та привілеї
  • Еліт-користувач - максимальні привілеї, ексклюзивний контент
  • Адміністратор - управління обліковими записами, моніторинг транзакцій

2.3 Зовнішні залежності

  • Google Auth API - автентифікація через Google-акаунт
  • PayPal API - платіжні транзакції та виведення коштів
  • Спортивний дата-провайдер API - актуальні дані подій та результати
  • ШІ-модель - асистент для рекомендацій (бажана інтеграція)

4. Нефункціональні вимоги

4.1 Продуктивність

  • NFR-P-01: Час відповіді для 95% операцій < 3 с при 500 000 одночасних користувачів.
  • NFR-P-02: Система витримує пікове навантаження без деградації сервісу.

4.2 Безпека

  • NFR-S-01: Обов'язкова автентифікація для доступу до фінансових операцій та ставок.
  • NFR-S-02: Усі персональні та фінансові дані зберігаються у зашифрованому вигляді (AES-256).
  • NFR-S-03: HTTPS обов'язковий для всіх з'єднань (TLS 1.2+).
  • NFR-S-04: Відповідність GDPR: користувач має право на видалення своїх даних.
  • NFR-S-05: Доступ до сервісу тільки для повнолітніх (18+), верифікація при реєстрації.

4.3 Зручність використання

  • NFR-U-01: Кросплатформенний адаптивний інтерфейс: підтримка Mobile та Desktop (320px–2560px).
  • NFR-U-02: Ключові сценарії (реєстрація, ставка, виведення) доступні не більше ніж у 3 кліки.

4.4 Надійність

  • NFR-R-01: Uptime ≥ 99.9% (24/7), включно з критичними годинами спортивних трансляцій.
  • NFR-R-02: Автоматичний резервний backup бази даних кожні 24 години.
  • NFR-R-03: Транзакції виведення коштів є атомарними — часткове списання без зарахування неприпустиме.

4.5 Масштабованість

  • NFR-SC-01: Хмарна архітектура підтримує горизонтальне масштабування при зростанні навантаження.
  • NFR-SC-02: Система здатна збільшити потужність вдвічі без суттєвих змін архітектури.

5. Обмеження

  • C-01: Front-end реалізується на React.
  • C-02: Розгортання - хмарна інфраструктура (cloud-native).
  • C-03: Обов'язкове шифрування даних на рівні бази та транспортному рівні.
  • C-04: Бюджет проєкту: 80 000 EUR.

  • C-07: Відповідність вимогам GDPR щодо збору та обробки персональних даних.
  • C-08: Геолокаційні обмеження: доступ до беттингу відповідно до локального законодавства.
